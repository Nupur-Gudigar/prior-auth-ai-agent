# HIPAA-Aware Design Documentation

## Overview
This project processes synthetic patient data through a 
Bronze/Silver/Gold Medallion pipeline with HIPAA-aware 
de-identification at the Silver layer.

## De-identification Approach
We use Microsoft Presidio to automatically detect and redact 
Protected Health Information (PHI) from patient records.

### What gets removed at Silver layer:
| Field | Bronze (Raw) | Silver (De-identified) |
|---|---|---|
| First Name | "James" | `<PERSON>` |
| Last Name | "Smith" | `<PERSON>` |
| SSN | "123-45-6789" | `<US_SSN>` |
| City | "Louisville" | `<LOCATION>` |
| Birthdate | "1970-03-15" | `<DATE_TIME>` |

### What is retained:
- Patient UUID (random, contains no PHI)
- Medical conditions, medications, procedures
- Gender, race (used for bias analysis)

## Audit Trail
Every AI agent decision is logged to a Delta table with:
- Timestamp
- Patient ID
- Decision (APPROVED/DENIED)
- Policy citation
- Reasoning steps

This provides full compliance traceability required in 
production healthcare AI systems.

## Responsible AI Disclosure
All patient data used in this project is synthetic, 
generated by Synthea (MITRE Corporation).
No real Protected Health Information was used at any stage.

## Tools Used
- **Microsoft Presidio** — PHI detection and redaction
- **Delta Lake** — ACID-compliant audit trail
- **Databricks** — HIPAA-eligible cloud platform
- **Synthea** — Synthetic patient data generator
